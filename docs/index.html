Open-source control room for coding agents

面向编码 Agent 的开源控制台

Gitdex

Stop babysitting AI coding in chat. Gitdex gives your agent team a control room: PM, Planner, Dev, QA, Review, and Merge agents work in sequence or parallel while GitHub stores the trail.

别再陪 AI 在聊天里绕圈。Gitdex 给你的 agent 团队一个控制台：PM、Planner、Dev、QA、Review、Merge 可以串行或并行协作，GitHub 负责保存痕迹。

Install Gitdex 安装 Gitdex View GitHub 查看 GitHub

Why this exists

为什么需要它

Chat is not a delivery system

聊天不是交付系统

AI agents can write patches fast. The mess starts when they need to coordinate: who owns the next step, who validates, who reviews, who rebases, and what context survives.

AI agent 写 patch 很快。混乱出现在协作阶段：谁负责下一步、谁验证、谁审核、谁 rebase、哪些上下文能留下来。

No more lost requirements

需求不再蒸发

The PM agent shapes messy intent into executable work. GitHub stores the resulting spec so every later agent reads the same source.

PM agent 把混乱想法整理成可执行任务。GitHub 保存产出的规格，让后续每个 agent 读取同一个来源。

Agents leave evidence

Agent 必须留证据

Dev, QA, Review, and Merge agents produce logs, comments, labels, and PRs. GitHub is the ledger; agents are the actors.

Dev、QA、Review、Merge agent 产生日志、评论、标签和 PR。GitHub 是账本，agent 才是行动者。

State survives the session

状态不依赖会话

Agents can stop, retry, or hand off. The durable state lives in GitHub labels, comments, issues, and PRs, so the next agent can continue.

agent 可以停止、重试、交接。持久状态保存在 GitHub 标签、评论、issue 和 PR 中，下一个 agent 可以接着干。

Agent-native execution loop

Agent 原生执行闭环

Make agents work like a team, not a pile of prompts

让 Agent 像团队，而不是一堆 Prompt

Gitdex gives every agent a role, every role a control point, and every control point a durable record. GitHub is the storage layer; the agent workflow is the product.

Gitdex 给每个 agent 一个角色，给每个角色一个控制点，给每个控制点一条持久记录。GitHub 是存储层，agent 工作流才是产品主体。

01

Clarify

澄清需求

The PM agent helps shape a rough request into a clear requirement before any GitHub issue is created.

PM agent 先帮助用户把模糊想法澄清成明确需求，再进入 GitHub issue。

02

Plan issues

拆分 issue

The planner creates GitHub issues with owned paths, dependencies, acceptance criteria, and whether work can run in parallel.

Planner 创建 GitHub issue，明确 owned paths、依赖、验收标准，以及哪些工作可以并行。

03

Develop and QA

开发与 QA

Developer agents open PRs from isolated worktrees; QA agents validate test coverage and user-visible behavior before review.

Developer agent 在隔离 worktree 中创建 PR；QA agent 在 review 前验证测试覆盖和用户可见行为。

04

Review and merge

审核与合并

Reviewer and merge agents check PR readiness, route rebase or spec blockers, merge approved work, and close the issue.

Reviewer 和 Merge agent 检查 PR 就绪状态，处理 rebase 或规格阻塞，合并通过的工作并关闭 issue。

What you get

你得到什么

An agent system you can actually debug

一个真的能调试的 Agent 系统

Agents first, GitHub persisted

Agent 优先，GitHub 持久化

The workflow is agent-driven, but the memory is durable: specs, QA failures, reviews, and merge results are stored where engineers already look.

流程由 agent 驱动，但记忆是持久的：规格、QA 失败、审核和合并结果都保存在工程师本来就会看的地方。

Agents with jobs

Agent 有明确岗位

Planner plans, Dev builds, QA breaks things, Review challenges the diff, Merge closes the loop.

Planner 负责拆解，Dev 负责实现，QA 负责找问题，Review 负责挑战 diff，Merge 负责闭环。

Watch the machine run

看得见机器在跑

See which agent is working, which issue it owns, how long it has run, and the raw execution output.

看到哪个 agent 在跑、负责哪个 issue、跑了多久，以及原始执行输出。

Auto-run without magic

自动推进，但不装神秘

Gitdex runs anything runnable, pauses when blocked, and keeps the controls visible instead of hiding automation behind a spinner.

Gitdex 会推进所有可运行任务，遇到阻塞就停下，并把控制点暴露出来，而不是把自动化藏在转圈里。

Runs where your repo lives

运行在你的仓库旁边

Local data, local worktrees, your GitHub CLI, your Codex login. No black-box cloud queue required.

本地数据、本地 worktree、你的 GitHub CLI、你的 Codex 登录。不需要黑盒云队列。

Hackable by default

默认可折腾

Install the CLI, run dev mode, wire it as a service, fork the workflow, change the prompts, ship your own control room.

装 CLI、跑 dev mode、注册服务、fork 工作流、改 prompt，做你自己的控制台。

Stable release

稳定版本

Install Gitdex v0.2.0

安装 Gitdex v0.2.0

One command pulls the stable v0.2.0 tag, installs dependencies, builds Gitdex, and links the gitdex CLI into ~/.local/bin.

一条命令拉取稳定的 v0.2.0 tag，安装依赖、构建 Gitdex，并把 gitdex CLI 链接到 ~/.local/bin。
