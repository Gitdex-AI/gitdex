From requirement to merge

从需求到合并

A visible path for agent work

让 Agent 工作路径清晰可见

Gitdex separates product discovery, GitHub issue execution, and operations follow-up while letting multiple issues move in parallel.

Gitdex 将产品澄清、GitHub issue 执行和运营跟进分开，同时允许多个 issue 在依赖允许时并行推进。

01

Plan

规划

Chat with the PM agent, confirm the requirement, then let the planner create scoped GitHub issues.

先与 PM agent 澄清需求，确认后由 Planner 创建范围明确的 GitHub issue。

02

Build

开发

Developer agents work in isolated worktrees, create PRs, and include tests for the accepted behavior.

Developer agent 在隔离 worktree 中实现需求、创建 PR，并补充覆盖验收行为的测试。

03

Validate

验证

QA agents verify tests, run browser checks on assigned preview ports, and send failures back to the right stage.

QA agent 检查测试覆盖，在指定预览端口验证浏览器行为，并把失败退回正确阶段。

04

Review and merge

审核与合并

Reviewer and merge agents inspect PR readiness, handle rebase requirements, and close issues after merge.

Reviewer 和 Merge agent 检查 PR 是否就绪、处理 rebase 要求，并在合并后关闭 issue。

Auto-run orchestration

自动推进所有可运行 issue，也支持不同 issue 在不同阶段并行运行。

Gitdex 运行在你的机器上，使用本地数据、本地 worktree、本地 GitHub CLI 和 Codex 登录状态。
